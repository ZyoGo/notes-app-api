======================================== TDD feature Users from Music API ========================================

•	Skenario 1: Adding User (memasukan user baru)
    o	Request URL
        •	Method: POST
        •	Path: /users
        •	Body:
            •	username: string
            •	password: string
            •	fullname: string
    o	Spesifikasi Pengujian:
        •	Pastikan response memiliki status code 201.
        •	Pastikan header response Content-Type memiliki nilai application/json; charset=utf-8.
        •	Pastikan body response adalah object.
        •	Pastikan body response memiliki properti dan nilai yang sesuai.
        •	Pastikan data pada response body memiliki userId dan nilainya tidak kosong.


•	Skenario 2: Adding User with Exist Username (memasukan user baru dengan username yang sudah ada)
    o	Request URL
        •	Method: POST
        •	Path: /users
        •	Body:
            •	username: string (exist username)
            •	password: string
            •	fullname: string
    o	Spesifikasi Pengujian:
        •	Pastikan response memiliki status code 400.
        •	Pastikan header response Content-Type memiliki nilai application/json; charset=utf-8.
        •	Pastikan body response adalah object.
        •	Pastikan body response memiliki properti dan nilai yang sesuai.


•	Skenario 3: Adding User with Bad User Payload (memasukan user baru dengan data yang tidak sesuai)
    o	Request URL
        •	Method: POST
        •	Path: /users
        •	Body: Dynamically set Bad User Payload
    o	Spesifikasi Pengujian:
        •	Pastikan response memiliki status code 400.
        •	Pastikan header response Content-Type memiliki nilai application/json; charset=utf-8.
        •	Pastikan body response adalah object.
        •	Pastikan body response memiliki properti dan nilai yang sesuai.


•	Skenario 4: Getting User by Correct Id (mendapatkan user menggunakan id)
    o	Request URL
        •	Method: GET
        •	Path: /users/{id}
    o	Spesifikasi Pengujian:
        •	Pastikan response memiliki status code 200.
        •	Pastikan header response Content-Type memiliki nilai application/json; charset=utf-8.
        •	Pastikan body response adalah object.
        •	Pastikan body response memiliki properti dan nilai yang sesuai.
        •	Pastikan data pada response body memiliki properti user yang merupakan sebuah objek.
        •	Pastikan objek user di dalam data hanya memiliki properti id, username, dan fullname dengan nilai yang sesuai.


•	Skenario 5: Getting User by Incorrect Id (mendapatkan user menggunakan id yang salah)
    o	Request URL
        •	Method: GET
        •	Path: /users/{id}
    o	Spesifikasi Pengujian:
        •	Pastikan response memiliki status code 404.
        •	Pastikan header response Content-Type memiliki nilai application/json; charset=utf-8.
        •	Pastikan body response adalah object.
        •	Pastikan body response memiliki properti dan nilai yang sesuai.
